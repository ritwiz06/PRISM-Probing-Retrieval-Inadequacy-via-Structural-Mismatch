## RAS_V4 Summary

RAS_V4 models joint adequacy for each backend:

- route adequacy from RAS_V3-style query and routing features
- evidence adequacy from top-k evidence diagnostics
- expected answerability through a learned linear adequacy score

Evidence features include score gaps, overlap with query terms, identifier/title matches, source diversity, redundancy, KG/path completeness, hybrid agreement, semantic consistency, distractor/negation contamination, public-document noise, and answerability markers. All features are computed without gold labels at inference time.

## Rescue-Comparison Summary

RAS_V4 alone does not close the calibrated rescue gap. RAS_V4 plus rescue reaches `0.958`, matching calibrated rescue. This supports the conclusion that route-and-evidence adequacy and top-k answer rescue are complementary: evidence-quality scoring improves diagnostics and explanation, but the post-route rescue overlay still recovers answer-bearing evidence that route selection alone does not exploit.

## Promotion-Decision Summary

RAS_V4 remains analysis-only. It is interpretable and preserves curated, external, generalization_v2, and public raw guardrails, but it does not materially improve adversarial answer accuracy over production computed RAS. The production router remains `computed_ras`.

## Known Limitations

- RAS_V4 uses a compact linear model and engineered evidence features; this is interpretable but may underfit hard route-boundary cases.
- Evidence adequacy is estimated from retrieved text and scores, not from human support labels.
- Human alignment is approximate because RAS_V4 outputs were not directly included in the original comparative packet.
- Calibrated rescue remains stronger for adversarial answer accuracy, showing that answer-layer recovery still matters.
- The model is not promoted to production without stronger adversarial answer gains.
