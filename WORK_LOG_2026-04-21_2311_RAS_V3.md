# WORK LOG: RAS V3 Framework

Timestamp: 2026-04-21 23:11

## Summary

Implemented an additive RAS_V3 framework for PRISM. RAS_V3 formalizes routing as an interpretable feature-based linear adequacy model over the four route families: BM25, Dense, KG, and Hybrid. It trains learned route weights from curated plus dev-only data, saves the learned model as JSON, produces per-route scores and feature contributions, compares against router baselines, connects to existing human-comparative preferences, and writes report-facing artifacts.

Production routing remains unchanged. Computed RAS remains the production router because RAS_V3 improves adversarial route accuracy but does not improve adversarial answer accuracy over computed RAS and does not beat calibrated rescue.

## RAS_V3 Summary

RAS_V3 defines a fixed feature vector and learned linear route weights. The feature vector includes:

- identifier-heavy signal
- lexical exactness signal
- semantic abstraction signal
- deductive cue signal
- relational bridge signal
- multi-hop cue signal
- negation/contrast signal
- route ambiguity signal
- computed RAS route and margin signals
- keyword-router route signals
- source/corpus-type signals
- query-local graph availability signal
- public-document noise signal
- top-k rescue opportunity signal for analysis

The learned model is a balanced multinomial logistic regression over these transparent features. It writes weights, intercepts, and protocol metadata to `data/eval/ras_v3_model.json`.

## Promotion-Decision Summary

Decision: keep RAS_V3 analysis-only.

Reason:

- adversarial route accuracy improves over computed RAS
- adversarial answer accuracy does not improve over computed RAS
- calibrated rescue still has higher adversarial answer accuracy
- human route-alignment is not worse but also not better
- guardrail benchmarks do not regress

Production router remains `computed_ras`.

## Known Limitations

- RAS_V3 uses small local/dev training data.
- The model is interpretable, but learned weights can still overfit benchmark construction patterns.
- Human comparison is post-hoc route alignment, not direct annotation of RAS_V3 outputs.
- RAS_V3 improves route accuracy on adversarial test but does not fix answer/evidence limitations by itself.
- Calibrated rescue remains stronger because it includes post-retrieval top-k evidence rescue, not only route scoring.
