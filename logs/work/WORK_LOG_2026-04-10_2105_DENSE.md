## Concepts and Topics Used

- Dense retrieval: retrieval based on vector similarity rather than exact token overlap.
- Embedding: a numeric vector representation of a text chunk or query.
- Chunking: splitting documents into smaller text units so retrieval can return focused evidence.
- Cosine similarity: similarity measure used by normalized vectors in the numpy fallback.
- FAISS: a vector index library; supported behind the interface but not active here.
- SentenceTransformers: local embedding model library; optional path kept available for `all-MiniLM-L6-v2`.
- Fallback backend: a deterministic numpy implementation used when heavier embedding/index dependencies are missing.
- Semantic benchmark slice: a query set designed around paraphrase and meaning rather than exact lexical overlap.
- Hit@k: whether the gold evidence appears anywhere in the top `k` results.
- Top-1: whether the first retrieved item is the gold evidence.
- Dense-vs-BM25 comparison: verifies that Dense performs better than BM25 on paraphrastic semantic queries.
