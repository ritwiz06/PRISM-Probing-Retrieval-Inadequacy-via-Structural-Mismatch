## Known Limitations

- The hard benchmark is hand-constructed and intentionally stress-heavy; it diagnoses failure modes rather than estimating real-world frequency.
- Answer correctness still uses normalized string matching, not human grading.
- Failure buckets are rule-assisted and inspectable, not manually adjudicated labels.
- The classifier router is only an analysis baseline and does not replace computed RAS in production.
- Computed RAS is no longer best on this adversarial benchmark; that result is preserved in artifacts rather than hidden.
- Public lexical arbitration is useful for some public raw cases but did not improve this broader hard-case set, so it remains analysis-only.
