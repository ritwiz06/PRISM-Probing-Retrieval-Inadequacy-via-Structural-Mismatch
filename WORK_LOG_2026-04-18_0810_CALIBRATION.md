## Calibration Summary

- Calibration is optional and analysis-only.
- Computed RAS remains the production router.
- The calibrator starts from computed RAS and only overrides in narrow hard-case conditions.
- Signals used include:
  - RAS route and margin,
  - keyword router route,
  - classifier router route/probabilities,
  - identifier-heavy query detection,
  - misleading exact-term cues,
  - semantic cue words,
  - explicit KG-negation wording,
  - deductive and relational cue patterns.
- Tuning protocol:
  - adversarial dev split only,
  - route accuracy as the tuning objective,
  - adversarial test held out,
  - existing benchmarks used only as sanity checks.
- Selected profile: `classifier_heavier`.

## Top-k Rescue Summary

- Top-k rescue is optional and analysis-only.
- It does not fetch new evidence and does not use gold labels.
- It only reorders evidence already retrieved in top-k.
- It scores candidate evidence with:
  - query token overlap,
  - identifier matches,
  - penalties for evidence dominated by negated/distractor query spans,
  - a small backend/source consistency bonus.
- It is gated so it only applies when an explicit negated or distractor span is detected and a replacement evidence item has a clear score margin.
- On adversarial test, top-k rescue alone improved answer accuracy from `0.917` to `0.958`.

## Known Limitations

- Calibration is tuned on a small hand-built adversarial dev split, so the selected thresholds should not be treated as globally optimal.
- The classifier router is still a local analysis baseline and should not replace computed RAS without broader validation.
- Top-k rescue improves answer accuracy partly by better use of already-retrieved evidence, not by improving route selection.
- Failure-delta buckets reuse rule-assisted labels from the adversarial failure-analysis layer; they are diagnostic, not human adjudications.
- The calibrated/rescued path remains optional because it adds interpretability complexity compared with plain computed RAS.
